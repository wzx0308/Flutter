## 一、钱包充值 & 支付宝支付

### 1.1 双重支付确认机制

#### What - 是什么

双重支付确认机制是一种**支付结果可靠到达**的技术方案，通过**前端轮询**和**支付宝异步回调**两条通道同时监听支付结果，确保支付状态最终一致。

#### Why - 为什么需要

单通道确认存在风险：
- 仅依赖轮询：用户关闭页面后无法得知支付结果
- 仅依赖回调：网络不稳定时回调可能丢失

双重确认确保**至少一条通道**能收到支付结果，提升支付成功率和用户体验。

#### Where - 在哪里使用

- 用户扫码支付后，前端展示二维码并启动轮询
- 后端同时接收支付宝异步回调
- 两条通道互为备份，确保支付状态最终一致

#### How - 如何实现

**后端核心代码**（`wallet.service.ts`）：

```typescript
// 轮询确认支付
async queryAndConfirmOrder(tradeNo: string) {
  // 1. 先查本地数据库，已成功直接返回
  const transaction = await this.prisma.walletTransaction.findUnique({
    where: { tradeNo },
  });
  if (transaction?.status === 'SUCCESS') {
    return { status: 'SUCCESS', paid: true };
  }

  // 2. 调用支付宝查询
  const queryResult = await this.alipayService.queryTrade(tradeNo);
  const tradeStatus = (queryResult as any)?.tradeStatus;

  // 3. 支付成功，执行Prisma事务更新余额
  if (tradeStatus === 'TRADE_SUCCESS' || tradeStatus === 'TRADE_FINISHED') {
    await this.prisma.$transaction(async (tx) => {
      await tx.walletTransaction.update({
        where: { tradeNo },
        data: { status: 'SUCCESS', alipayTradeNo: (queryResult as any).tradeNo },
      });
      await tx.user.update({
        where: { id: transaction.userId },
        data: { balance: { increment: transaction.amount } },
      });
    });
    return { status: 'SUCCESS', paid: true };
  }

  return { status: tradeStatus || 'WAIT_BUYER_PAY', paid: false };
}
```

**异步回调处理**（`wallet.service.ts`）：

```typescript
async handleAlipayNotify(params: Record<string, string>) {
  const tradeStatus = params.trade_status;
  const outTradeNo = params.out_trade_no;

  // 1. RSA2验签
  if (!this.alipayService.verifyNotify(params)) {
    this.logger.warn(`回调验签失败: ${outTradeNo}`);
    return 'failure';
  }

  // 2. 幂等性检查
  const transaction = await this.prisma.walletTransaction.findUnique({
    where: { tradeNo: outTradeNo },
  });
  if (!transaction || transaction.status === 'SUCCESS') {
    return 'success'; // 已处理，直接返回成功
  }

  // 3. 金额校验（误差<0.01）
  const totalAmount = parseFloat(params.total_amount);
  if (Math.abs(totalAmount - Number(transaction.amount)) > 0.01) {
    this.logger.warn(`金额不匹配: ${totalAmount} vs ${transaction.amount}`);
    return 'failure';
  }

  // 4. 事务更新余额
  if (tradeStatus === 'TRADE_SUCCESS' || tradeStatus === 'TRADE_FINISHED') {
    await this.prisma.$transaction(async (tx) => {
      await tx.walletTransaction.update({
        where: { tradeNo: outTradeNo },
        data: { status: 'SUCCESS', alipayTradeNo: params.trade_no },
      });
      await tx.user.update({
        where: { id: transaction.userId },
        data: { balance: { increment: transaction.amount } },
      });
    });
  }

  return 'success';
}
```

### 1.2 行锁防超扣 & 幂等性

#### What - 是什么

- **行锁防超扣**：使用数据库行级锁 `SELECT ... FOR UPDATE`，在并发场景下锁定用户行，防止余额被重复扣减
- **幂等性**：通过 `idempotencyKey` 确保同一笔转账请求即使重复提交也只执行一次

#### Why - 为什么需要

并发转账场景：
- 用户快速点击两次"确认转账"
- 网络抖动导致前端重试
- 没有锁和幂等性，可能导致**余额超扣**或**重复转账**

#### Where - 在哪里使用

- 用户发起转账时
- 余额扣减操作

#### How - 如何实现

**行锁 + 事务核心代码**（`transfer.service.ts`）：

```typescript
async createTransfer(userId: string, dto: CreateTransferDto) {
  const { receiverId, amount, paymentPassword, idempotencyKey } = dto;

  // 1. 基础校验
  if (userId === receiverId) {
    throw new BadRequestException('不能给自己转账');
  }

  // 2. 验证支付密码
  const sender = await this.prisma.user.findUnique({
    where: { id: userId },
    select: { paymentPassword: true },
  });
  if (!sender.paymentPassword) {
    throw new BadRequestException('请先设置支付密码');
  }
  const passwordValid = bcrypt.compareSync(paymentPassword, sender.paymentPassword);
  if (!passwordValid) {
    throw new UnauthorizedException('支付密码错误');
  }

  // 3. 数据库事务：行锁 + 扣款 + 加钱
  const result = await this.prisma.$transaction(async (tx) => {
    // 行锁防超扣
    const senderForUpdate = await tx.$queryRaw`
      SELECT id, balance FROM users WHERE id = ${userId}::uuid FOR UPDATE
    `;
    const currentBalance = Number((senderForUpdate as any[])[0].balance);
    if (currentBalance < amount) {
      throw new BadRequestException(`余额不足，当前余额 ¥${currentBalance.toFixed(2)}`);
    }

    // 幂等性检查
    if (idempotencyKey) {
      const existing = await tx.transfer.findFirst({
        where: { senderId: userId, remark: `__idem:${idempotencyKey}` },
      });
      if (existing) {
        return existing; // 已执行过，直接返回
      }
    }

    // 扣款
    await tx.user.update({
      where: { id: userId },
      data: { balance: { decrement: amount } },
    });

    // 加钱
    await tx.user.update({
      where: { id: receiverId },
      data: { balance: { increment: amount } },
    });

    // 创建转账记录
    const transfer = await tx.transfer.create({
      data: {
        senderId: userId,
        receiverId,
        amount,
        status: 'PENDING',
        remark: `__idem:${idempotencyKey}`, // 幂等性标记
        expireAt: new Date(Date.now() + 24 * 60 * 60 * 1000),
      },
    });

    return transfer;
  });

  return result;
}
```

### 1.4 24小时自动退回

#### What - 是什么

定时任务每5分钟扫描过期转账（超过24小时未确认收款），自动执行退回操作，将资金返还给发送方。

#### Why - 为什么需要

- 转账有有效期（24小时）
- 超时未收款需要自动退回
- 避免资金长期挂账

#### Where - 在哪里使用

- 后台定时任务 `@Cron(EVERY_5_MINUTES)`
- 扫描 `status === PENDING && expireAt < now()` 的转账

#### How - 如何实现

```typescript
@Cron(CronExpression.EVERY_5_MINUTES)
async handleExpiredTransfers() {
  const expiredTransfers = await this.prisma.transfer.findMany({
    where: {
      status: 'PENDING',
      expireAt: { lt: new Date() },
    },
  });

  for (const transfer of expiredTransfers) {
    try {
      await this.refundTransfer(transfer.id);
      this.logger.log(`自动退回转账: ${transfer.id}`);
    } catch (error) {
      this.logger.error(`退回失败: ${transfer.id}`, error);
    }
  }
}

async refundTransfer(transferId: string) {
  return this.prisma.$transaction(async (tx) => {
    const transfer = await tx.transfer.findUnique({ where: { id: transferId } });

    // 收款方扣钱
    await tx.user.update({
      where: { id: transfer.receiverId },
      data: { balance: { decrement: transfer.amount } },
    });

    // 发送方加钱
    await tx.user.update({
      where: { id: transfer.senderId },
      data: { balance: { increment: transfer.amount } },
    });

    // 更新转账状态
    await tx.transfer.update({
      where: { id: transferId },
      data: { status: 'REFUNDED' },
    });

    // 创建双向REFUND流水
    await tx.walletTransaction.createMany({
      data: [
        {
          userId: transfer.receiverId,
          amount: -transfer.amount,
          type: 'REFUND',
          status: 'SUCCESS',
          subject: `转账退回`,
        },
        {
          userId: transfer.senderId,
          amount: transfer.amount,
          type: 'REFUND',
          status: 'SUCCESS',
          subject: `转账退回`,
        },
      ],
    });

    return transfer;
  });
}
```

### 1.5 支付宝SDK深度封装

#### What - 是什么

对支付宝官方SDK `alipay-sdk@^4.14.0` 进行二次封装，提供简洁的支付接口，隐藏底层复杂性。

#### Why - 为什么需要

- 官方SDK API复杂，直接使用代码冗余
- 需要统一管理配置（appId、密钥、网关）
- 需要封装验签逻辑

#### Where - 在哪里使用

- 钱包充值模块
- 支付状态查询
- 回调验签

#### How - 如何实现

```typescript
// alipay.service.ts
@Injectable()
export class AlipayService {
  private readonly sdk: InstanceType<typeof AlipaySdk>;

  constructor(private config: ConfigService) {
    this.sdk = new AlipaySdk({
      appId: this.config.get('ALIPAY_APP_ID')!,
      privateKey: this.config.get('ALIPAY_PRIVATE_KEY')!,
      alipayPublicKey: this.config.get('ALIPAY_PUBLIC_KEY'),
      gateway: this.config.get('ALIPAY_GATEWAY'),
      signType: 'RSA2',
    });
  }

  // 预下单（返回二维码URL）
  async createTradePayOrder(params: {
    tradeNo: string;
    amount: string;
    subject: string;
  }) {
    return this.sdk.exec('alipay.trade.precreate', {
      bizContent: {
        out_trade_no: params.tradeNo,
        total_amount: params.amount,
        subject: params.subject,
      },
      notifyUrl: this.config.get('ALIPAY_NOTIFY_URL'),
    });
  }

  // 查询交易状态
  async queryTrade(tradeNo: string) {
    return this.sdk.exec('alipay.trade.query', {
      bizContent: { out_trade_no: tradeNo },
    });
  }

  // 验证回调签名
  verifyNotify(params: Record<string, string>): boolean {
    return this.sdk.checkNotifySign(params);
  }
}
```

### 2.1 Socket.IO实时通信

#### What - 是什么

基于Socket.IO的实时通信系统，支持双向消息推送、用户在线状态、正在输入状态等实时交互。

#### Why - 为什么需要

- 聊天需要低延迟消息推送
- HTTP轮询效率低、延迟高
- 需要实时感知对方在线状态

#### Where - 在哪里使用

- 单聊/群聊消息收发
- 用户上下线通知
- 正在输入状态
- 已读回执
- 通话信令

#### How - 如何实现

**后端Gateway**（`chat.gateway.ts`）：

```typescript
@WebSocketGateway({ cors: { origin: '*' }, namespace: '/chat' })
export class ChatGateway implements OnGatewayConnection, OnGatewayDisconnect {
  @WebSocketServer()
  server: Server;

  private onlineUsers = new Map<string, string>(); // userId -> socketId

  async handleConnection(client: Socket) {
    // JWT认证
    const token = client.handshake.auth?.token;
    const payload = await this.jwtService.verifyAsync(token);
    const userId = payload.sub;

    // 记录在线状态
    this.onlineUsers.set(userId, client.id);
    client.join(`user:${userId}`);
    this.server.emit('user:status', { userId, online: true });
  }

  @SubscribeMessage('chat:send')
  async handleSend(client: Socket, data: { conversationId: string; content: string }) {
    const senderId = client.data.userId;

    // 互相关注检查
    const { allowed } = await this.chatService.canSendMessage(data.conversationId, senderId);
    if (!allowed) {
      client.emit('chat:restricted', { message: '互相关注后可继续聊天' });
      return;
    }

    // 保存消息
    const message = await this.chatService.saveMessage(
      data.conversationId, senderId, data.content
    );

    // 推送给所有会话成员
    const conv = await this.prisma.conversation.findUnique({
      where: { id: data.conversationId },
      include: { members: true },
    });

    for (const member of conv.members) {
      this.server.to(`user:${member.userId}`).emit('chat:receive', {
        id: message.id,
        conversationId: data.conversationId,
        senderId,
        content: message.content,
        type: message.type,
        createdAt: message.createdAt,
      });
    }
  }
}
```

```typescript
async canSendMessage(conversationId: string, senderId: string) {
  const conv = await this.prisma.conversation.findUnique({
    where: { id: conversationId },
    include: { members: true },
  });

  // 单聊才检查
  if (conv.type !== 'PRIVATE') {
    return { allowed: true };
  }

  const otherMember = conv.members.find(m => m.userId !== senderId);
  if (!otherMember) {
    return { allowed: false };
  }

  // 检查是否互相关注
  const [isFollowing, isFollowed] = await Promise.all([
    this.prisma.follow.findFirst({
      where: { followerId: senderId, followingId: otherMember.userId },
    }),
    this.prisma.follow.findFirst({
      where: { followerId: otherMember.userId, followingId: senderId },
    }),
  ]);

  const isMutualFollow = isFollowing && isFollowed;

  if (!isMutualFollow) {
    // 检查是否已发送过消息
    const messageCount = await this.prisma.message.count({
      where: { conversationId, senderId },
    });
    if (messageCount > 0) {
      return { allowed: false, reason: '互相关注后可继续聊天' };
    }
  }

  return { allowed: true };
}
```

const MAX_ROUNDS = 8;           // 保留最近8轮对话
const SUMMARIZE_THRESHOLD = 20; // 超过20条消息触发摘要
const SUMMARIZE_BATCH = 8;      // 每次摘要8条

async saveMessage(userId: string, conversationId: string, role: string, content: string) {
  const record = await this.prisma.conversationMemory.findFirst({
    where: { userId, conversationId },
  });

  const newMessage = { role, content, timestamp: new Date().toISOString() };

  if (!record) {
    await this.prisma.conversationMemory.create({
      data: { userId, conversationId, messages: [newMessage] },
    });
    return;
  }

  const messages = (record.messages as any[]) || [];
  messages.push(newMessage);

  // 超过阈值时触发摘要
  if (messages.length > SUMMARIZE_THRESHOLD) {
    const toSummarize = messages.splice(0, SUMMARIZE_BATCH);
    const summary = await this.summarize(record.summary, toSummarize);

    await this.prisma.conversationMemory.update({
      where: { id: record.id },
      data: {
        messages: messages.slice(-MAX_RECENT_MESSAGES),
        summary,
      },
    });
  }
}

async summarize(existingSummary: string | null, messages: any[]): Promise<string> {
  const prompt = existingSummary
    ? `请将以下对话与已有摘要合并，生成新的摘要（不超过200字）：\n已有摘要：${existingSummary}\n\n新对话：${JSON.stringify(messages)}`
    : `请为以下对话生成摘要（不超过200字）：\n${JSON.stringify(messages)}`;

  const response = await this.llm.invoke([new HumanMessage(prompt)]);
  return response.content as string;
}

**向量检索**（`rag.service.ts`）：

```typescript
const CHUNK_SIZE = 500;
const CHUNK_OVERLAP = 50;
const DEFAULT_TOP_K = 5;

async indexDocument(sourceType: string, sourceId: string, content: string) {
  // 1. 文档分块
  const splitter = new RecursiveCharacterTextSplitter({
    chunkSize: CHUNK_SIZE,
    chunkOverlap: CHUNK_OVERLAP,
    separators: ['\n\n', '\n', '。', '！', '？', '.', '!', '?', ' ', ''],
  });
  const chunks = await splitter.splitText(content);

  // 2. 向量化
  const embeddings = await this.embeddings.embedDocuments(chunks);

  // 3. 存储到数据库
  for (let i = 0; i < chunks.length; i++) {
    await this.prisma.documentChunk.create({
      data: {
        sourceType,
        sourceId,
        content: chunks[i],
        embedding: embeddings[i],
      },
    });
  }
}

async search(query: string, topK: number = DEFAULT_TOP_K) {
  // 1. 查询向量化
  const queryEmbedding = await this.embeddings.embedQuery(query);

  // 2. 余弦相似度计算（PostgreSQL扩展）
  const chunks = await this.prisma.$queryRaw`
    SELECT id, content,
      1 - (embedding <=> ${queryEmbedding}::vector) as similarity
    FROM document_chunks
    WHERE embedding IS NOT NULL
    ORDER BY similarity DESC
    LIMIT ${topK}
  `;

  return chunks;
}
```

使用Access Token + Refresh Token双token机制，Access Token短期有效，Refresh Token长期有效。

#### Why - 为什么需要

- Access Token过期后无需重新登录
- 提升用户体验
- 安全性与便利性平衡

#### Where - 在哪里使用

- 用户登录
- 接口鉴权
- Token刷新

#### How - 如何实现

```typescript
// auth.service.ts
async login(user: User) {
  const payload = { sub: user.id, username: user.username };

  // Access Token: 2小时有效
  const accessToken = this.jwtService.sign(payload, { expiresIn: '2h' });

  // Refresh Token: 7天有效
  const refreshToken = this.jwtService.sign(payload, { expiresIn: '7d' });

  // 存储Refresh Token
  await this.prisma.user.update({
    where: { id: user.id },
    data: { refreshToken },
  });

  return { accessToken, refreshToken };
}

async refreshTokens(refreshToken: string) {
  const payload = this.jwtService.verify(refreshToken);
  const user = await this.prisma.user.findUnique({ where: { id: payload.sub } });

  if (user.refreshToken !== refreshToken) {
    throw new UnauthorizedException('Invalid refresh token');
  }

  const newAccessToken = this.jwtService.sign(
    { sub: user.id, username: user.username },
    { expiresIn: '2h' }
  );

  return { accessToken: newAccessToken };
}
```
